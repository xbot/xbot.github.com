NGINX不允许向静态文件提交POST方式的请求，否则报405错误。测试方法为，使用curl向服务器上的静态文件提交POST请求：

{% codeblock lang:bash %} curl -d 1=1 http://localhost/version.txt {% endcodeblock %} 得到以下结果： {% codeblock lang:html %} 405 Not Allowed

405 Not Allowed

nginx/1.0.11
{% endcodeblock %}

网上传抄的添加以下配置的解决办法不可用：

{% codeblock lang:nginx %} error_page 405 =200 @405; location @405 { root /srv/http; } {% endcodeblock %}

一种不完美但可用的方法为：

{% codeblock lang:nginx %} upstream static_backend { server localhost:80; } server { listen 80; # ... error_page 405 =200 @405; location @405 { root /srv/http; proxy_method GET; proxy_pass http://static_backend; } } {% endcodeblock %}

即转换静态文件接收的POST请求到GET方式。
